# 🔐 Security Policy

## Supported Versions

The following versions of CoxyInsure are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| 0.x.x   | :white_check_mark: |
| < 0.x   | :x:                |

> We strongly recommend always using the latest version of the protocol and frontend to ensure maximum security.

---

## 🚨 Reporting a Vulnerability

We take the security of **CoxyInsure** very seriously.  
If you discover a vulnerability, please report it responsibly.

### 📧 How to Report

Send an email to:

**support@coxygen.co**

Include the following details:

- Description of the vulnerability  
- Steps to reproduce the issue  
- Potential impact (what could go wrong)  
- Any suggested fixes (if available)  
- Screenshots or proof of concept (if applicable)  

---

## ⏱️ Response Time

- **Acknowledgement:** Within 24–48 hours  
- **Initial assessment:** Within 3–5 days  
- **Resolution timeline:** Depends on severity  

We will keep you updated throughout the process.

---

## 🛠️ What to Expect

If your report is accepted:

- The issue will be investigated and patched  
- You may be credited for the discovery (if desired)  
- A security update or advisory may be released  

If your report is declined:

- We will provide a clear explanation  
- You may be asked for additional details  

---

## 🔒 Responsible Disclosure

To protect users and the protocol:

- Do **not** publicly disclose the vulnerability before it is fixed  
- Do **not** exploit the issue for personal gain  
- Do **not** access data that does not belong to you  

---

## ⚠️ Scope of Security

This project includes:

- Smart contracts (Plutus)
- Frontend interfaces
- Backend APIs
- Database systems

Security vulnerabilities may include:

- Smart contract flaws  
- Unauthorized fund access  
- Authentication or authorization issues  
- Data exposure  
- Transaction manipulation  

---

## 🧠 Security Best Practices (For Contributors)

- Never expose private keys or sensitive credentials  
- Validate all inputs (frontend & backend)  
- Follow secure smart contract design principles  
- Use proper access control for admin functions  
- Test thoroughly before deployment  

---

## 🚀 Commitment

CoxyInsure is committed to maintaining a **secure, reliable, and trustworthy decentralized insurance system**.

We appreciate your efforts in helping us improve the security of the protocol 🛡️
