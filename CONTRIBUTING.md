# 🤝 Contributing to CoxyInsure

First off, thank you for your interest in contributing to **CoxyInsure** 🚀  
We welcome contributions from developers, designers, researchers, and anyone passionate about decentralized finance and blockchain innovation.

This guide will help you understand how to contribute effectively to the project.

---

## 🌍 Our Philosophy

CoxyInsure is built as a **collaborative, open-source DeFi protocol**.  
We believe in:

- Transparency
- Community-driven development
- Security-first engineering
- Continuous learning and improvement

---

## 📌 Before You Start

Please make sure you:

- Read our [Code of Conduct](CODE_OF_CONDUCT.md)
- Understand the project structure
- Check existing issues and discussions before creating new ones

---

## 🛠️ Ways to Contribute

### 💻 Code Contributions
- Fix bugs
- Improve performance
- Add new features
- Refactor existing code

---

### 📄 Documentation
- Improve README or guides
- Add tutorials or walkthroughs
- Fix typos or unclear explanations

---

### 🧪 Testing
- Report bugs
- Suggest improvements
- Test features on different environments

---

### 🎨 UI/UX Improvements
- Improve layout and responsiveness
- Enhance user experience
- Suggest design upgrades

---

## ⚙️ Development Setup

```bash
git clone https://github.com/your-username/coxyinsure.git
cd coxyinsure
````

### Backend

* Configure database (`db.php`)
* Import SQL schema
* Start PHP server

### Frontend

* Open `index.html` or `main.html`
* Connect wallet (Cardano testnet)

---

## 🔄 Contribution Workflow

1. **Fork the repository**
2. **Create a new branch**

   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
4. **Commit clearly**

   ```bash
   git commit -m "Add: short description of change"
   ```
5. **Push to your fork**

   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open a Pull Request**

---

## 🧾 Pull Request Guidelines

* Keep PRs **focused and small**
* Provide a **clear description**
* Link related issues (if any)
* Include screenshots for UI changes
* Ensure your code does not break existing features

---

## 🐞 Reporting Bugs

When reporting a bug, include:

* Clear description of the issue
* Steps to reproduce
* Expected vs actual behavior
* Screenshots (if applicable)

---

## 💡 Feature Requests

We welcome ideas!
When suggesting features:

* Explain the problem it solves
* Describe your proposed solution
* Keep it aligned with the project vision

---

## 🔐 Security Guidelines

Because this is a **DeFi protocol**, security is critical:

* Do not expose private keys or sensitive data
* Follow best practices when interacting with smart contracts
* Report vulnerabilities responsibly

---

## 🧠 Coding Standards

* Write clean and readable code
* Use consistent naming conventions
* Add comments where necessary
* Avoid unnecessary complexity

---

## ⚖️ Governance Awareness

CoxyInsure includes a governance system:

* Changes affecting claims or execution logic should be reviewed carefully
* Ensure compatibility with smart contract logic
